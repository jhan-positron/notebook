## Model Arithmetic

The predictions used measured same-machine SNC-OFF data plus topology
arithmetic, not `goal.md` targets.

L3 local-die estimate:

```text
SNC-OFF local L3 average = 60.535 ns
Hop model = local, +24 ns, +48 ns across the three dies
SNC3 local die ~= 60.535 - (0 + 24 + 48) / 3
               ~= 36.535 ns
Measured local 4 MiB = 35.892 ns
```

DRAM node estimate:

```text
SNC-OFF local-socket DRAM prior = 137.728 ns
SNC3 local DRAM ~= 137.728 - 24 = 113.728 ns
Measured local DRAM = 112.751 ns
```

Bandwidth ceiling:

```text
DDR5-6400 channel = 6400 MT/s * 8 B = 51.2 GB/s
One SNC3 die = 4 channels = 204.8 GB/s
Socket = 12 channels = 614.4 GB/s

Measured single-node DRAM peak = 204.238 GB/s
Measured socket read endpoint = 608.882 GB/s
```

## Future Project Summary

SNC3 is beneficial when the workload can exploit locality: local L3 and local
DRAM latency improve materially, and per-node memory binding gives predictable
one-die behavior. SNC3 is harmful if a workload binds many threads to one SNC
memory node and expects whole-socket bandwidth: it plateaus at about 204 GB/s
instead of the roughly 600 GB/s whole-socket capability. When memory is placed
locally per die with `--local`, peak socket bandwidth is not lost; the full
socket read endpoint reaches 608.882 GB/s.

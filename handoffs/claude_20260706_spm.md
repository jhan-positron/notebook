    memory-mapped via the CMN mesh into 4x AXI4 256-bit slave ports;
    writes 16/32 B granular, internal reads always a full 2048-bit word;
    (2) metadata is PUSH, not poll — SPM delivers each 46 B record to the
    ARM core chosen by acp_dsu_sel via ACP, landing coherently in cache;
    (3) registers via CMN SBSX -> 64-bit AXI4-Lite -> spm_axil_regs.
  - Metadata = routing data = RG data (Routing + Generated); 46 B (368 b)
    per data word in 384-bit rows; "Generated" = SVL top-k/sum/max.
  - Metadata NOT memory-mapped for reads; ad-hoc reads only via indirect
    AXI-Lite debug regs. Metadata writes from CMN ride the AXI port but
    the store address comes from the in-payload spm_addr, not the AXI
    address.
  - CMN-side masters: 10 C1-Nano cores (5 DSUs), 4 SPM DMA engines, host
    CPU via PCIe.
  - Geometry: word 2048 b (256 B), 64 Ki words; 16-bit word address
    ([3:0] bank, [5:4] segment, [15:6] row); CMN sees a 24-bit
    byte-addressed 16 MiB window. Endianness unspecified in both docs —
    little-endian assumed, flagged for Bly/Wen.
  - Transparent to SW: banking/tiles/segments, 4-cycle turn scheduling,
    credits, bank-conflict retry, store-and-forward vs bypass. Not

## Gotchas & decisions

- SYSTEM_CONFIG env var silently overrides runtron --instance — scrub it
  (plus TRON_LOG_LEVEL/SPDLOG_LEVEL) before any benchmark on this host.
- /var/tmp and /tmp are HOST-LOCAL on the delphi machines; /scratch is the
  shared NFS filer (same paths on every delphi host). Build in /var/tmp for
  speed, publish to /scratch for cross-host use.
- Cross-host binaries: a tron build's RPATH points into its build tree —
  running the copy from /scratch on another host needs LD_LIBRARY_PATH to
  the copied lib dirs. Smoke-test (ldd + --version) ON THE TARGET HOST; a
  pass on the build host proves nothing.
- Grant rungs are discrete (measured, ALLCORE doc sections 2.5-2.6): fast
  cores get 4400 only when ALL other cores are <=2700; any other shape gets
  4100; there is no 4200 rung. Realized worker frequency under every fast
  shape: ~4030-4060 MHz mean.
- HT siblings share one clock (max 3 MHz delta measured across all 144
  pairs): per-logical-CPU CLOS association cannot split a physical core —
  keep siblings in the same class (the tooling auto-adds them). Fast + idle
  sibling is fine (and slightly faster than fast + busy sibling).
- Boot default = 16 fixed PCT physical cores fast (8 per package, same
  positions on both hosts), all else clamped; only 8 of TRON's 80 app cores
  fall in the fast set — hence the ~90%-clamped signature.
- Turbostat busy-core means are only comparable across shapes after
  filtering to the SAME core role set — the grant ladder rewards
  lightly-threaded cores, so moving helper threads between classes shifts
  the aggregate mean without touching the workers.
- PR-3070 changes C++ code, not just the resource map — testing it needs a
  build from pull/3070/head; the binary dir's own config/resource-map.yaml
  drives thread placement at runtime.
- The benchmark runners' kill cannot reap the root-owned turbostat child —
  orphans accumulate (contaminating later per-cpu TSVs unless analyses are
  snapshot-bounded); clean up with sudo pkill turbostat.
- checkerboard's process-safety check pgreps full cmdlines for
  runtron/rinzler/run-*sweep*/run-benchmark keywords — monitoring processes
  must not carry those strings in their command lines.
- Memlock: host hard limit is 197,971,044 KB, below checkerboard's 200 GB
  default; set CHECKERBOARD_MEMLOCK_KB=197971044.

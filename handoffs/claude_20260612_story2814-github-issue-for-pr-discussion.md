## Gotchas & decisions
- **Over-counting trap (the key technical insight in the issue draft):** after PR #2906, `gof::populated` is reset by `free_staging_if_idle()` (gof.hpp:235), so at the backfill site `!populated` is also true for every GOF that WAS transferred and then had its staging evicted — the common case the PR deliberately creates. The counter needs a persistent discriminator surviving eviction, e.g. an `ever_transferred` bit set on first `inflight_guard::commit()`, not a `populated` check at `prepare_for_dma()` time.
- Naming: the gal member is `pages` (`page_info pages[4]`, gof.hpp:255); `gal::page_info` is wrong (it's the element type).
- Resolving review threads does NOT clear a CHANGES_REQUESTED review decision on GitHub; only a new review or dismissal does. That, not CI, is what shows the PR as "blocked".

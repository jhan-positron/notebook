## Gotchas & decisions
- core-power enable silently wipes CLOS3's boot clamp config, and turbo-freq enable --auto wipes all four CLOS configs and rewrites the whole partition — v2 never calls core-power enable and re-pins CLOS0/CLOS3 on every apply; revert verifies clos3-max=2700.
- Control-plane pinning: --min 800 --max 2000 starves to ~0.8 GHz under contention; must use --min 2000 --max 2000.
- SST state does not survive reboot (BIOS reprograms the PCT CLOS partition every boot) — re-run flat_freq_apply after any reboot.
- sudoers NOPASSWD paths differ: /opt/intel-speed-select/intel-speed-select on 3bda, the workspace build on 3af6; the user's wrapper precheck fails non-interactively — v2 auto-detects the binary and honors a FLAT_FREQ_ISST env override.
- Scripts scp'd from Windows carry CRLF; every upload was followed by sed -i 's/\r$//' before execution.
- TF-disabled 3900 cap comes from the fused TRL 24-active-cores bucket; with C6 disabled in 3bda's BIOS idle cores never park, so non-HP cores cannot see the 4100 bucket — the TF high-priority path bypasses TRL counting entirely, which is why TF-on wins.
- 4400 on arbitrary cores (TF --auto) is real but only grantable when all other cores sit at/below the 2700 clip — a power/aggregate loss (4x4.4 + 140x2.7 = 395.6 GHz-cores vs 470.8 flat), so it was rejected.

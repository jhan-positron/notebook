## Gotchas & decisions

- `SYSTEM_CONFIG` env var silently overrides runtron `--instance` — scrub it
  (plus `TRON_LOG_LEVEL`/`SPDLOG_LEVEL`) before any benchmark.
- checkerboard's process-safety check pgreps full cmdlines for
  runtron/rinzler/run-*sweep*/run-benchmark keywords — monitoring processes
  must not carry those strings in their command line (a `bash -c` monitor
  with "runtron" in an echo aborted a run non-interactively).
- Memlock: host hard limit is 197,971,044 KB, below checkerboard's 200 GB
  default → set `CHECKERBOARD_MEMLOCK_KB=197971044`.
- Hugepage preflight compares CONFIGURED pages (not free) and tolerates
  same-user `libpos` leftovers (runtron reuses same-geometry slice files);
  foreign-owned leftovers in your slice range hard-fail the run — check
  `fuser` before removing (group positron has the perms).
- `run-multi-sweep.sh` length lists (-p/-g/--shared-system-prompt-lengths)
  are ZIPPED by position, not cross-multiplied; instances auto-derive from
  TP (tp4→2, tp2→4, tp1→8 on this 8-FPGA host).
- FPGA BAR wedge: if any of
  `/sys/bus/pci/devices/0000:{10,13,38,3b,90,93,b9,bc}:00.0/resource` has
  fewer than 3 non-zero BAR lines, do NOT benchmark. Fix: `pho bw images use
  -c <slot> archer_agm_01.05.08.00.rbf` for unconfigured cards, then warm
  reboot (images must be resident at POST). `pho accel reset` cannot fix
  cards missing from the USB↔PCI map (chicken-and-egg). PCI remove+rescan
  alone cannot work — the BIOS-sized stack MMIO windows are immutable at
  runtime.
- `intel-speed-select` writes results to stderr; on 3bda only
  `/opt/intel-speed-select/intel-speed-select` is NOPASSWD.
- Flat-freq verification is two-layer: CLOS readback (config) + turbostat
  busy-core Bzy_MHz distribution (measured). Clamp signature = busy cores
  pinned 2600-2799 MHz.
